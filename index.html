📡MULTIPLE ACCESS CONTROL

FDMA · TDMA · CDMA

Channel access methods that allow multiple users to share the same communication medium — the foundation of all wireless networks.

FDMA — Frequency TDMA — Time CDMA — Code Multiple Access
Concept Overview

What is Multiple Access?

Multiple access techniques allow many users to simultaneously share a finite radio frequency spectrum.

🌐 The Core Idea

In any wireless system, the available bandwidth is limited. Multiple Access techniques divide this shared resource — either by frequency, time, or code — so many users can communicate simultaneously without interfering with each other. These techniques form the backbone of all cellular, satellite, and wireless networks.

🎵 FDMA

Frequency Division Multiple Access divides the available spectrum into sub-bands. Each user is assigned a unique frequency channel for the entire call duration. Like different radio stations — each on its own frequency.

⏱ TDMA

Time Division Multiple Access gives each user the full bandwidth, but only during their assigned time slot. Like taking turns speaking — everyone uses the same frequency but at different times.

🔑 CDMA

Code Division Multiple Access lets all users transmit simultaneously on the same frequency using unique orthogonal codes. Like people speaking different languages in the same room — each listener only understands their own language.

FDMA

Frequency Division Multiple Access

Definition, working, and step-by-step example

📌 Definition

FDMA is a channel access method where the total available frequency bandwidth is divided into non-overlapping sub-bands (channels). Each user is permanently assigned one channel for the duration of their communication session.

⚙️ Step-by-Step Working

  1. Total Bandwidth AllocationThe system starts with a total bandwidth B (e.g., 25 MHz) divided into N channels, each of width W = B/N (e.g., 200 kHz).
  2. Guard BandsSmall guard bands are placed between channels to prevent inter-channel interference (crosstalk).
  3. Channel AssignmentWhen User 1 initiates a call, they are assigned Channel 1 (f₁). User 2 gets Channel 2 (f₂), and so on.
  4. Simultaneous TransmissionAll users transmit simultaneously, each on their own frequency. Bandpass filters at receivers extract only the desired channel.
  5. Channel ReleaseWhen the call ends, the channel is released and can be re-assigned to a new user.

🔢 Numerical Example

// FDMA Example — 3 users, total BW = 30 MHz
Total Bandwidth (B) = 30 MHz
Number of users (N) = 3
Guard Band (G) = 500 kHz

Channel Width (W) = (B − (N−1)×G) / N
W = (30 − 2×0.5) / 3 = 29/3 ≈ 9.67 MHz

User 1 → 1 to 9.67 MHz (f₁ center = 4.835 MHz)
User 2 → 10.17 to 19.84 MHz (f₂ center = 15.005 MHz)
User 3 → 20.34 to 30 MHz (f₃ center = 25.17 MHz)

✅ Purpose & Advantages

• Simple to implement
• No timing synchronization needed
• Continuous transmission per user
• Mature, proven technology (1G cellular)

📡 Real-World Usage

• 1G cellular (AMPS — Advanced Mobile Phone System)
• AM/FM Radio broadcasting
• Satellite transponders
• Cable TV (each channel on its own frequency)

TDMA

Time Division Multiple Access

Definition, working, and step-by-step example

📌 Definition

TDMA is a channel access method where all users share the same frequency channel but transmit in sequential time slots within a repeating frame. Each user is assigned one or more specific time slots per frame.

  1. Frame StructureTime is divided into frames. Each frame has a fixed duration T (e.g., 40 ms). Each frame is divided into N equal time slots (e.g., 8 slots).
  2. Slot AssignmentUser 1 → Slot 1, User 2 → Slot 2 ... User N → Slot N. The assignment repeats every frame.
  3. Burst TransmissionUsers compress their data and transmit in rapid bursts during their allocated slot. Voice at 64 kbps is stored and burst at 8× speed during the slot.
  4. Guard TimesSmall guard times separate slots to account for propagation delays and clock differences.
  5. SynchronizationBase station broadcasts sync signals. All mobile users synchronize their slot timing to the base station.
// TDMA Example — GSM Standard
Frame duration (T) = 4.615 ms
Slots per frame (N) = 8
Slot duration = T/N = 4.615/8 = 0.577 ms

Data rate per slot = 270.833 kbps × (1/8) = 33.85 kbps
Guard time per slot = 30.46 µs
Users per channel = 8

✅ Purpose & Advantages

• More efficient spectrum use than FDMA
• Can dynamically allocate multiple slots to one user
• Digital — supports data + voice
• Foundation of 2G GSM networks

📡 Real-World Usage

• 2G GSM (Global System for Mobile)
• IS-136 (D-AMPS)
• Digital Enhanced Cordless Telecom (DECT)
• Satellite systems (INTELSAT)

CDMA

Code Division Multiple Access

Definition, working, and step-by-step example with spreading codes

📌 Definition

CDMA uses spread-spectrum technology where each user is assigned a unique pseudo-random code (chip sequence). All users transmit simultaneously on the same frequency, but their signals can be separated at the receiver using the unique codes — based on orthogonality of chip sequences.

  1. Assign Orthogonal Chip CodesEach user gets a unique chip code of length N (e.g., N=4). Codes are orthogonal: dot product of any two different codes = 0.
  2. Signal EncodingUser's data bit (1 or 0) is XOR'd with their chip code. Bit 1 → send chip code as-is. Bit 0 → send inverted chip code (complement).
  3. TransmissionAll encoded signals are summed and transmitted on the same frequency channel simultaneously.
  4. Signal Separation (Decoding)Receiver multiplies received signal with desired user's chip code, then sums all chips. Due to orthogonality, other users' signals sum to 0, and the desired signal is recovered.
  5. Data RecoveryIf sum / N = +1 → data bit was 1. If sum / N = -1 → data bit was 0.
// CDMA Example — 2 users, chip length N=4
User A code: [ +1, −1, +1, +1 ] Data bit: 1
User B code: [ +1, +1, −1, +1 ] Data bit: 0

// Encoded signals
A encoded: +1×[+1,−1,+1,+1] = [ +1, −1, +1, +1 ]
B encoded: −1×[+1,+1,−1,+1] = [ −1, −1, +1, −1 ]

// Channel sum (what's transmitted)
Sum: [ 0, −2, +2, 0 ]

// Decoding for User A
Multiply: [+1,−1,+1,+1] × [0,−2,+2,0] = [0,+2,+2,0]
Sum = 4, Divide by N=4 → +1 → Data bit = 1 ✓

✅ Purpose & Advantages

• All users share same frequency — max efficiency
• Inherently secure (need correct code)
• Resistance to multipath fading
• Soft handoff capability
• 3G standard foundation

📡 Real-World Usage

• 3G WCDMA/UMTS
• cdmaOne (IS-95) and CDMA2000
• GPS satellite positioning system
• 4G LTE uses OFDMA (evolution of CDMA)
• Military secure communications
